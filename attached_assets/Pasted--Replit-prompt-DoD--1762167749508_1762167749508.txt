▶️ 给 Replit 的最终执行工单（可直接粘贴）

标题：启用证据包轮转 & Evidence Archive 管理页（生产上线执行）

范围（不得越界修改）
	•	不改主链路：银行解析/7态状态机/UploadResponse/熔断/支持矩阵。
	•	本次仅涉及：证据包 私有目录、Admin 受控下载/删除/轮转、审计记录、/admin/evidence 管理页与 i18n。
	•	“已正常工作的功能”坚决不改动。

⸻

1) 环境与权限（必须）

目标：配置 Secrets，确保私有目录与 RBAC 生效。

操作
	1.	设置环境变量（Secrets）
	•	EVIDENCE_ROTATION_DAYS=30
	•	EVIDENCE_KEEP_MONTHLY=1
	•	TASK_SECRET_TOKEN=<按现有值>
	2.	确认私有目录存在且可写：evidence_bundles/（不在 /static 下）。
	3.	确认 Admin 账号可登录；非 Admin 访问上述路由返回 401/403。

验收输出
	•	截图：Secrets 配置页
	•	目录清单：evidence_bundles/ 存在且权限正确

⸻

2) 路由与RBAC联调（必须）

目标：所有操作强制 Admin + 审计日志。

需验证的路由（全为 Admin-only）
	•	GET /admin/evidence（管理页）
	•	GET /downloads/evidence/list（列出现有 bundles）
	•	GET /downloads/evidence/latest（下载最新 ZIP）
	•	GET /downloads/evidence/file/<filename>（按文件名下载）
	•	POST /downloads/evidence/delete（删除单个 bundle）
	•	POST /tasks/evidence/rotate（触发轮转）— 需 Header：X-TASK-TOKEN: <token>

验收输出
	•	以 Admin 调用成功；以 非Admin/无 Token 调用失败（401/403）
	•	audit_logs 表新增 evidence_download / evidence_delete / evidence_rotation 记录（含 IP、UA、详情）

⸻

3) 前端管理页与 i18n（必须）

目标：/admin/evidence 页面显示与操作完整、双语无硬编码。

检查点
	•	表格列：文件名 / 创建时间 / 大小 / SHA256 / 来源 / 操作（下载、删除、轮转按钮）
	•	i18n 键齐全（中/英切换正确）
	•	UI 色板严格：Hot Pink #FF007F、Dark Purple #322446、Black #000000

验收输出
	•	页面截图（中/英各一张）
	•	删除与轮转的弹窗文案为 i18n 字段（非硬编码）

⸻

4) 轮转策略联调（必须）

目标：30 天留存 + 每月第一个包长期保留；审计全量记录。

操作
	•	先 手动生成 至少 2 个证据包（已有功能：下载/打包触发）
	•	调用：

POST /tasks/evidence/rotate
Header: X-TASK-TOKEN: <token>


	•	观察返回：{"success": true, "kept":[...], "deleted":[...]}

验收输出
	•	轮转返回 JSON（原样）
	•	audit_logs 内新增 evidence_rotation 记录（含 kept/deleted 明细）

⸻

5) 完整性校验（必须）

目标：下载 ZIP 后校验 manifest.json 中的 SHA256 与实际文件一致。

操作
	1.	GET /downloads/evidence/latest 下载 ZIP
	2.	解压并执行 SHA256 校验（任选其一）
	•	macOS/Linux：shasum -a 256 <file>
	•	Windows（PowerShell）：Get-FileHash <file> -Algorithm SHA256
	3.	比对 manifest.json 中的 sha256 值

验收输出
	•	1 个文件的 命令结果 与 manifest.json 值一致的截图/文本

⸻

6) 审计抽查SQL（必须）

操作（提供查询结果文本）

-- 最近 50 条证据相关操作
SELECT action, ip_address, user_agent, reason, created_at
FROM audit_logs
WHERE action IN ('evidence_download','evidence_delete','evidence_rotation')
ORDER BY created_at DESC
LIMIT 50;

-- 最近 5 次轮转详情
SELECT *
FROM audit_logs
WHERE action='evidence_rotation'
ORDER BY created_at DESC
LIMIT 5;

验收输出
	•	两条 SQL 的结果文本（打码敏感信息即可）
