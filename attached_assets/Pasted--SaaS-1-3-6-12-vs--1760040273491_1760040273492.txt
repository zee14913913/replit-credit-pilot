端到端自动化（你可照抄到 Zapier/Make/n8n）
以下全部无需写代码；我已把详细步骤写在 ZIP 里的 Automation_Recipes.md，这里给你“执行摘要”＋字段映射：




1) 供应商 PO → 自动开票 → 回发给供应商
触发：
* Gmail/Outlook：主题包含“PO”或附件是 PDF/XLSX/CSV
* 或用 Google Form/Typeform 收 PO
动作：
1. 解析 PO（供应商、金额、税、到期日、PO 编号）。
2. Invoices 追加一行：
    * Type=Purchase，Related PO ID=<PO号>，Entity ID=<Supplier ID>，Subtotal/Tax/Total，Due Date，Status=Draft。
3. 用 Docs 模板生成 PDF 发票 → 邮件/WhatsApp 给 Suppliers.Billing Email。
4. Tasks 创建 Request GRN（收货单）任务，Due Date=PO+3 天。




2) POS → 销售入账 → 现金流
触发：
* POS Webhook（推荐）或每天导入 CSV 至 Google Drive 文件夹
动作：
1. 逐笔写入 POS_Sales（含 Invoice No、支付方式、Items JSON）。
2. 同步写入 Cashflow：Category=Sales，Direction=In，Ref Table=POS_Sales，Ref ID=<Sale ID>。
3. 可选：按门店/品类聚合日结。




3) 银行对账（CSV） → Bank_Transactions → 自动匹配
触发：
* Drive 新增银行 CSV/PDF（PDF 用 OCR 节点先转表格）
动作：
1. 逐笔写入 Bank_Transactions。
2. 匹配逻辑：
    * 入账：找 POS_Sales.Net Amount 或 Invoices(Type=Sales).Total 金额±容差（如 RM 1）且日期±3天；匹配成功 → Reconciled=Y，Match Ref ID 写来源单号。
    * 出账：匹配 Invoices(Type=Purchase) 或 Payroll.Net Pay。
3. 失败留 Task：银行对账未匹配。



4) 月度工资（薪资跑批）
触发：
* 每月 1 日（或任意日期）
动作：
1. 计算 Net Pay = Base + OT + Commission + Allowances - Deductions - EPF - SOCSO - EIS。
2. Payroll 追加；为每位员工创建支付 Task，并可邮件工资单（可用 Docs 模板 + Gmail 节点）。
3. 写 Cashflow：Category=Payroll，Direction=Out。





5) 税务报表（月/季/年）
触发：
* 每月/每季/每年期末
动作：
1. Revenue：汇总 POS_Sales.Net Amount。
2. COGS：汇总 Invoices(Type=Purchase).Total（按品类/供应商）。
3. OPEX：工资 + 营销 + 租金 + 其它（来自 Cashflow）。
4. 计算：Gross Profit、EBITDA、Taxable Income、Estimated Tax。
5. 写 Tax_Report；导出 PDF 给会计/Audit。



计算与风控（核心公式/规则）
* 发票合计：Total = Subtotal + Tax + Shipping（Invoices）
* POS 净额：Net Amount = Gross - Discount + Tax（POS_Sales）
* 净发工资：Net Pay = Base + OT + Commission + Allowances - Deductions - EPF - SOCSO - EIS（Payroll）
* 现金流：入＝销售/收款/退税；出＝采购/工资/租金/税金/贷款（Cashflow）
* 对账规则：金额容差 ±RM1、日期窗口 ±3 天、优先精确匹配发票号/参考号；匹配失败创建 Task
* 税务底线：所有数据留痕（来源、表、单号、日期），方便审计抽查；敏感账户号仅脱敏存储




如何把“两个店面 + 多产品线 + 分租/买卖”一并纳入
* 门店维度：POS_Sales.Store ID/Store Name、Cashflow.Store ID、Marketing_Posts.Store ID
* 产品线维度：在 POS_Sales.Items JSON 中携带 category（Lighting/Appliances/Beauty/Reno）以便统计；或新增一个 Products 维表（可选）。
* 分租/买卖：当生成合同/分租单据时，作为 Invoices(Type=Sales) 记录，Subcategory=Rental/Property；自动提醒合同到期（Tasks）。
