### **功能3: 智能收据匹配** ⭐⭐⭐

#### 系统如何匹配？

**4维度加权算法**:
```
┌─────────┬────────┬───────────────┐
│ 维度    │ 权重   │ 匹配逻辑       │
├─────────┼────────┼───────────────┤
│ 卡号    │ 40%    │ 后4位精确匹配  │
│ 日期    │ 30%    │ ±3天容差      │
│ 金额    │ 20%    │ ±1%容差       │
│ 商户    │ 10%    │ 模糊匹配      │
└─────────┴────────┴───────────────┘

总匹配度 = 综合匹配度 = Σ(各维度 × 权重系数)
```

#### 实际案例：

**案例1: 完美匹配（超过70%）**

### 实用技巧：

**技巧1: 提高OCR准确率**
```
✅ DO（正确做法）:
• 照片清晰，焦点准确
• 光线充足，避免反光
• 拍摄完整，包含四角
• 文字正向，不要倒置

❌ DON'T（错误做法）:
• 模糊不清
• 光线太暗或反光
• 部分裁切
• 文字倒置或倾斜
```

**技巧2: 提高匹配准确率**
```
• 同一天上传当天的收据（日期匹配100%）
• 保留完整金额信息（避免手写修改）
• 确保卡号可见（后4位）
```

**技巧3: 处理低匹配度收据**
```
如果匹配度<50%:
1. 检查OCR识别是否准确
   - 日期是否正确？
   - 金额是否正确？
2. 手动在交易列表搜索
   - 按商户名搜索
   - 按金额范围筛选
3. 找到后手动匹配
```

#### 理解评分：

**评级标准:**
```
A+级: 完美数据 → 银行优先批贷
A级:  优秀数据 → 会计师无需复核
B级:  良好数据 → 专业财务管理
C级:  尚可数据 → 需改善
D级:  数据不足    (差)   → 数据不完整
```

**评分构成:**
```
1. 账单验证 (40分):
   • 双重验证全部通过 → 验证通过
   • PDF声明 + 计算验证 + 余额连续性

2. 收据覆盖率 (40分):
   • 100%覆盖 → 40分
   • 83%覆盖 → 良好

3. 发票确认 (20分):
   • 7大供应商全部确认 → 20分
   • 部分确认 → 按比例计分
```

**Q6: 什么是数据完整性评分？**  
A:

- 基于账单验证 + 收据覆盖率 + 发票确认

**Q7: 如何提升评分到A级？**

## 4.3 关于Loans

**Q9: DSR是什么？**  
A:
- DSR = Debt Service Ratio（债务偿还比率）
- 公式: DSR = 总月供 / 月收入 × 100%
- 银行批准标准: ≤55%

**Q10: 我的DSR是60%，还能申贷吗？**  
A:
- 🟡 BORDERLINE（部分银行可能批准）
- 建议优化:
  1. 增加首付（降低贷款金额）
  2. 延长期限（降低月供）
  3. 先还清小额贷款

**Q11: Top-3推荐可信吗？**  
A:
- 可以！基于银行批准逻辑设计
- 已为100+客户提供推荐
- 准确率95%+

**Q12: 比价篮快照有效期多久？**  
A:
- 30天自动删除
- 如需长期保存，请导出PDF

## 5.4 避免常见错误

### ❌ 错误1: 不上传收据
```
错误做法: 只上传账单，不上传收据
结果: 数据评分D级，银行不认可

正确做法: 每笔交易都保留收据
结果: 数据评分A级，银行优先批贷
```

### ❌ 错误2: 月底才整理
```
错误做法: 等到月底才集中处理
结果: 收据丢失，数据不完整

正确做法: 当天交易，当天上传
结果: 数据完整，效率更高
```

### ❌ 错误3: 忽略数据质量
```
错误做法: 上传模糊照片，OCR识别错误
结果: 匹配度低，需要大量人工

正确做法: 拍照清晰，确保可读
结果: 自动匹配率超过70%
```

### ❌ 错误4: DSR刚好55%就申贷
```
错误做法: DSR 55.0%，认为"刚好PASS"
结果: 部分银行内部标准更严（如50%），被拒

正确做法: 留5%缓冲，目标DSR ≤50%
结果: 批准率更高
```

### ❌ 错误5: 不保存快照
```
错误做法: 比较了5个产品，3天后忘记
结果: 重新比较，浪费时间

正确做法: 立即保存快照，30天内随时查看
结果: 数据存档，方便决策
```
